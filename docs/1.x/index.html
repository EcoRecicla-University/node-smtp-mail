Introdução

A ideia deste projeto é ser um recurso rápido para envio de e-mail de maneira sistemática, a ser utilizado em pequenos projetos, não recomendado para ser utilizado em ambiente de produção..

Para ambiente de produção, recomenda-se a utilização de algum serviço de mensagens mais seguro e completo.

Instalação do sistema

Para iniciar, realize um fork deste projeto para sua conta, e, em seguida, realize o clone do repositório que foi criado em sua conta:

https://github.com/<seu-nome-de-usuario>/node-smtp-mail.git

ou via SSH (caso seja necessário):

git@github.com:<seu-nome-de-usuario>/node-smtp-mail.git

Com isso, você terá o projeto em sua máquina.

Antes de iniciar qualquer coisa no projeto, é importante instalar as dependências necessárias para o funcionamento do projeto. Para isso, utilize o gerenciador de pacotes npm. O comando para instalar as dependências é:

npm install

Com estas etapas concluídas, basta concluir os passos descritos na seção de Setup inicial para começar a utilizar o serviço.

Setup inicial

A etapa de configuração inicial deste serviço consiste em, basicamente, definir corretamente os arquivos e variáveis de ambiente padrão para que o projeto seja executado.

Com isso, o arquivo que necessita de alteração é o environment.js, localizado dentro do projeto, no caminho /src/env.

Inicialmente, você encontrará somente o arquivo de exemplo ( environment.example.js ). O arquivo definitivo de variáveis de ambiente será configurado logo abaixo. Para isso, siga os seguintes passos:

  1. No mesmo caminho do arquivo de exemplo ( /src/env ), crie um novo arquivo com o nome environment.js.
  2. Copie o conteúdo do arquivo de exemplo e cole no novo arquivo que você criou.

Para entender o que cada variável significa, você pode ler mais sobre environment. Dessa forma, você poderá definir os valores corretos de acordo com a sua necessidade.

Conclusão

Após ter concluído todas as etapas iniciais apresentadas nesta página, você está pronto para:

  • Desenvolver seus próprios template de e-mail, com as variáveis que desejar.
  • Configurar uma API para receber requisições e enviar e-mail.
  • Remover toda a lógica complexa de envio de e-mail da sua aplicação principal, onde a regra de negócio deve estar clara e simples.
